# Security and Sharing

## Safe To Publish

- this skill
- the bundled scripts and Remotion template
- documentation and example placeholder data

## Review Before Publishing

- generated `INSIGHTS.md`
- rendered posters and videos
- screenshots of the generated report

These outputs may reveal:

- project names
- client or company names
- work cadence and active dates
- preferred tools and workflows
- token volume and intensity

## Never Publish

- raw local Codex, Claude, Gemini, or Antigravity logs
- local sqlite files
- machine-specific debug artifacts
- absolute filesystem paths unless intentionally redacted

## If The User Wants A Public Portfolio Artifact

- rename sensitive projects
- summarize by category instead of exact repo names
- round metrics when precision is unnecessary
- confirm that dates and work patterns are okay to share
